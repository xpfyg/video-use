# 2026-07-11

## 重构 match_shots.py：按镜头拆分调用 + 本地合成 EDL

- 痛点：原脚本把整个 strategy + 整个 content_report 一次性塞进一个 prompt，过大易超长/不稳定。
- 新方案：对 strategy.shots 里每个 shot（beat）单独调一次模型，**每镜只传该镜台词 + label 匹配的素材内容**，全部跑完后在本地拼成完整 edl.json。
- 数据结构（已核实）：strategy 顶层 `shots[]` 每镜含 `shot_id/start/end/duration/text/beat/label`；`material_labels.json` 的 `materials[素材名].label` 与 strategy 的 label 同属 labels.json 体系 → 用 `shot.label == material.label` 过滤。`match_shots.py` 旧代码读 `strategy["templates"]` 是遗留死代码，已改为直接读 `strategy["shots"]`（兼容 `templates[...].shots`）。
- 新增函数：`load_material_labels` / `filter_candidates`（精确匹配；无匹配则按字符重叠度排「相近 label」素材作 fallback 并标 `label_fallback`）/ `build_candidate_content` / `match_one_shot`（桩掉可测）/ `dedup_ranges`（同素材区间重叠则本地后移）/ `assemble_edl`（本地拼 sources+ranges+total_duration_s）。
- 保留：`--use-ollama`(默认) / `--ollama-*` / `--ark-url` / `--ark-model`(默认仍 `DEFAULT_ARK_MODEL`) / `--grade` / `--audio-track` / `-o`；`run_ollama`/`run_deepseek`/`extract_json`/`load_json` 原样复用。
- 新增参数：`--material-labels`（默认 `<edit-dir>/material_labels.json`，缺失报错退出）、`--no-filter-label`（强制不分 label 用全部素材）。
- 验证：合成 fixture + monkeypatch `match_one_shot` 跑通 main()，断言 4 镜按 label 命中、fallback 命中最近标签、去重后移、total=16.0s 与 strategy 一致；`py_compile` 通过。**未跑真实 Ollama/ARK 端到端（无真实 edit-dir/网络），仅逻辑层验证。**
- 设计决策（与用户确认）：每镜头固定返回 1 段；无精确 label 匹配时用「相近 label」素材兜底。
