# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

`video-use` is a conversation-driven video editor skill for Claude Code. It enables editing videos through natural language — the LLM reads transcript/metadata, proposes a strategy, waits for user confirmation, then renders `final.mp4`.

This is a **customized fork** of `browser-use/video-use`, optimized for Chinese TTS dubbing workflows and snack-commerce short videos. The key difference from upstream: **no ElevenLabs dependency**. It natively consumes byte-dance TTS `metadata.json` (character-level timestamps) instead of Scribe transcription.

## Repository structure

- `helpers/` — 18 standalone Python scripts that do the actual work (transcribe, pack, render, grade, analyze, match). Each is invoked directly as `python helpers/<name>.py`.
- `skills/manim-video/` — Sub-skill for Manim animation overlays.
- `SKILL.md` — The production rules and editing craft document consumed by the agent at runtime.
- `install.md` — First-time setup instructions (clone, deps, ffmpeg, skill registration).
- `pyproject.toml` — Python dependencies (`requests`, `librosa`, `matplotlib`, `pillow`, `numpy`). Optional extras: `[animations]` (manim), `[snack_visual]` (opencv, imagehash, mlx-vlm), `[ollama]`.

## Development commands

```bash
# Install dependencies
pip install -e .
# or for snack-commerce workflows:
pip install -e ".[snack_visual]"

# Verify ffmpeg is available
ffprobe -version

# Run a helper (all helpers are standalone scripts)
python helpers/pack_metadata.py metadata.json
python helpers/render.py edit/edl.json -o edit/final.mp4
python helpers/grade.py input.mp4 -o graded.mp4 --preset warm_cinematic
python helpers/timeline_view.py input.mp4 0 10
```

There is no test suite, lint config, or build step beyond dependency installation.

## High-level architecture

### Two-layer reading model

The LLM never watches raw video. It reads through two layers:

1. **Packed transcript (`takes_packed.md`)** — always loaded. Phrase-level, time-annotated markdown. For TTS content this is generated by `pack_metadata.py` from `metadata.json`; for ASR content by `pack_transcripts.py` from `transcripts/*.json`.
2. **Visual composite (`timeline_view.py`)** — on-demand only. Produces a filmstrip + waveform + word-labels PNG for a specific time window. Used at decision points (ambiguous pauses, cut sanity checks, retake comparisons), not for continuous scanning.

### Render pipeline (hard rules)

`render.py` implements a specific 3-stage pipeline that must not be violated:

1. **Per-segment extract** — each EDL range is extracted with its color grade and 30ms audio fades (`afade=t=in:st=0:d=0.03,afade=t=out:st={dur-0.03}:d=0.03`) baked in.
2. **Lossless `-c copy` concat** — segments are concatenated without re-encoding into `base.mp4`.
3. **Overlays + subtitles LAST** — if overlays or subtitles exist, a single filtergraph composites animation overlays (with `setpts=PTS-STARTPTS+T/TB` so frame 0 lands at the overlay window start) and applies the `subtitles` filter **after** all overlays. Subtitles before overlays = silent failure (overlays hide captions).

### HDR handling

iPhone and many mirrorless cameras record HLG/PQ HDR in Rec.2020. `render.py` auto-detects HDR transfer characteristics and applies `zscale=tr=linear:npl=100,format=gbrpf32le,zscale=primaries=bt709,tonemap=hable,format=yuv420p` before the grade filter to avoid blown-out output.

### Grade system

`grade.py` supports named presets (`warm_cinematic`, `neutral_punch`, `snack_vivid`, `subtle`, `none`) and an `auto` mode that samples frames mathematically to emit bounded per-clip correction (capped at ±8%). Auto is the default when `render.py` receives `"grade": "auto"`. Presets are raw ffmpeg filter strings; custom filters can be passed directly.

### EDL format

`edl.json` is the contract between strategy and render:

```json
{
  "version": 1,
  "sources": {"C0103": "/abs/path/C0103.MP4"},
  "ranges": [{"source": "C0103", "start": 2.42, "end": 6.85, "beat": "HOOK", "quote": "...", "reason": "..."}],
  "grade": "warm_cinematic",
  "overlays": [{"file": "edit/animations/slot_1/render.mp4", "start_in_output": 0.0, "duration": 5.0}],
  "subtitles": "edit/master.srt",
  "total_duration_s": 87.4
}
```

`grade` can be a preset name, raw ffmpeg filter, or `"auto"`. `overlays` use paths relative to the EDL's directory or absolute. `subtitles` is optional and applied last.

### Session outputs

All outputs for a given editing session go into `<videos_dir>/edit/` — never inside the `video-use/` repo directory. Expected contents:

- `project.md` — appended session memory
- `takes_packed.md` — LLM reading view
- `edl.json` — cut decisions
- `clips_graded/` — per-segment extracts
- `master.srt` — output-timeline subtitles
- `animations/slot_<id>/` — per-animation source + render
- `verify/` — debug frames / timeline PNGs
- `preview.mp4` / `final.mp4`

### Snack-commerce workflow (audio-first)

For snack-product short videos on Apple Silicon, there is a fully local pipeline:

```
metadata.json/SRT
  → generate_strategy.py  → strategy.json + strategy.md
  → analyze_visual.py     → visual_report.json  (480p@1fps, CV quality scores)
  → analyze_content.py    → content_report.json + visual_catalog.md  (3s VLM windows)
  → match_shots.py        → edl.json  (text model matches shots to clips)
  → render.py --audio-track → final.mp4
```

`match_shots.py` sends the full strategy template + compressed visual catalog to a local text model in a single prompt, then refines the result deterministically to avoid repeated clips and guarantee valid durations.

### Animation slots

Animations are built in parallel sub-agents, one per slot. Each slot is a directory under `edit/animations/slot_<id>/`. Engines:

- **HyperFrames** — HTML/CSS/GSAP compositions; scaffold with `npx --yes hyperframes init .`
- **Remotion** — React compositions; scaffold with `npx create-video@latest`
- **Manim** — formal diagrams; see `skills/manim-video/SKILL.md`
- **PIL + PNG sequence + ffmpeg** — simple overlay cards

Each sub-agent receives a self-contained brief with absolute output path, resolution, fps, codec, palette, font path, frame-by-frame timeline, and deliverable checklist. Sub-agents have no parent context.

### TTS metadata format

The custom TTS workflow consumes byte-dance TTS `metadata.json`:

```json
{
  "summary": {"output": "dubbing.mp3", "speaker": "voice_id", "duration": 17.0},
  "events": [{"sentence": {"text": "...", "words": [{"word": "我", "startTime": 0.355, "endTime": 0.475}]}}]
}
```

Sentences with empty `words` arrays are skipped. Character-level timestamps are preferred over word-level ASR because they are millisecond-accurate with no drift.

## Important conventions

- **Chinese text**: no spaces between characters in subtitles or packed transcripts.
- **Silence threshold**: use `0.2s` for Chinese TTS content (default `0.5s` is too large for TTS sentence pauses).
- **Cut padding**: working window 30–200ms. Tighter for fast-paced short videos, looser for cinematic.
- **Self-eval cap**: maximum 3 render-evaluate loops before surfacing issues to the user.
- **No ElevenLabs API key required** for the TTS workflow; the original `transcribe.py` still calls ElevenLabs Scribe if used.
